Class: JMEPart

Numbas.parts.JMEPart

new JMEPart()

Judged Mathematical Expression

Student enters a string representing a mathematical expression, eg. x^2+x+1, and it is compared with the correct answer by evaluating over a range of values.

Properties:
Name Type Description
correctAnswerString jme

the definition of the correct answer, without variables substituted into it.
correctAnswer string

An expression representing the correct answer to the question. The student's answer should evaluate to the same value as this.
answerSimplificationString string

string from the XML defining which answer simplification rules to use
answerSimplification Array.<string>

names of simplification rules (see Numbas.jme.display.Rule) to use on the correct answer
checkingType string

method to compare answers. See Numbas.jme.checkingFunctions
checkingAccuracy number

accuracy threshold for checking. Exact definition depends on the checking type.
failureRate number

comparison failures allowed before we decide answers are different
vsetRangeStart number

lower bound on range of points to pick values from for variables in the answer expression
vsetRangeEnd number

upper bound on range of points to pick values from for variables in the answer expression
vsetRangePoints number

number of points to compare answers on
maxLength number

maximum length, in characters, of the student's answer. Note that the student's answer is cleaned up before checking length, so extra space or brackets aren't counted
maxLengthPC number

partial credit if the student's answer is too long
maxLengthMessage string

Message to add to marking feedback if the student's answer is too long
minLength number

minimum length, in characters, of the student's answer. Note that the student's answer is cleaned up before checking length, so extra space or brackets aren't counted
minLengthPC number

partial credit if the student's answer is too short
minLengthMessage string

message to add to the marking feedback if the student's answer is too short
mustHave Array.<string>

strings which must be present in the student's answer
mustHavePC number

partial credit to award if any must-have string is missing
mustHaveMessage string

message to add to the marking feedback if the student's answer is missing a must-have string.
mustHaveShowStrings boolean

tell the students which strings must be included in the marking feedback, if they're missing a must-have?
notAllowed Array.<string>

strings which must not be present in the student's answer
notAllowedPC number

partial credit to award if any not-allowed string is present
notAllowedMessage string

message to add to the marking feedback if the student's answer contains a not-allowed string.
notAllowedShowStrings boolean

tell the students which strings must not be included in the marking feedback, if they've used a not-allowed string?
